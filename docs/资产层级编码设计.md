# 资产层级编码设计（任意多层编码树）

**版本**：v1.0 ｜ **日期**：2026-06-12 ｜ 关联：需求文档、设计概要

## 目标

资产列表的分类层级支持**逐层构建对应编码并增删改查**，取代原只读的固定四层卡片导航。

## 数据模型

统一编码树 `assetTree`，节点结构：

- `id`：唯一标识（取完整编码）
- `name`：名称（如"三菱 MITSUBISHI"）
- `codeSeg`：本层编码段（如"MIT"）
- `code`：完整编码 = 父 code + "-" + codeSeg（顶层 = codeSeg）
- `children[]`：子分类节点（任意层级，可无限嵌套）
- `assets[]`：末端节点挂的资产实例（编号 = 节点 code + "-" + 流水号）

## 层级视图（资产列表）

- **逐层钻取** + 动态面包屑（可回跳任意层）
- **新增子分类**：填名称 + 编码段，实时预览完整编码；保存到当前节点 `children`
- **编辑节点**：改名称 / 编码段，编码变更**级联重算**子孙 `code`
- **删除节点**：有子节点或资产时二次确认（级联删除）
- **末端节点**：显示资产表 + 新增资产（编号自动拼接 = 节点 code + 流水号）

## 移除

原 `assetHierarchy`（categories/brands/series/assets 分离结构）及 `navigateToLevel2/3/4`、`asset-level2/3/4` 模板，由统一编码树 + 递归钻取取代（顺带修掉原面包屑中硬编码 "PLC" 的瑕疵）。

## 测试

逐层新增分类、编码拼接预览、编辑级联重算、删除级联、末端资产、响应式。
