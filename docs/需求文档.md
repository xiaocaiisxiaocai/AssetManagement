## 1. 项目概述

### 1.1 项目背景

部门现有资产管理依赖Excel手工记录，存在以下问题：
- 借还流程无审批控制，资产流向不清晰
- 缺乏统一编号规则，资产标识混乱
- 逾期归还无提醒机制，资产占用时间过长
- 操作记录无法追溯，责任难以界定
- 统计报表需手工整理，效率低且易出错

### 1.2 项目目标

构建部门级内网资产管理系统，实现：
- 资产全生命周期管理（入库、借出、归还、报废）
- 借还审批流程电子化，采用可配置工作流引擎（自由加节点、会签 / 或签、条件分支、加签 / 转签），支持代审
- 自动编号生成，支持按类别配置编号规则
- 逾期自动提醒，到期前提前通知
- 完整审计日志，所有操作可追溯
- 一键导出统计报表（Excel）

### 1.3 用户规模

- 预计用户总数：80-100人
- 日活跃用户：30-50人
- 峰值并发：<50人
- 部署环境：部门内网（无外网访问需求）

## 3. 功能需求

### 3.1 资产档案管理

#### FR-1.1 资产新增
- **描述**：资产管理员录入新资产信息
- **输入**：资产类别、名称、规格型号、品牌、数量、单价、供应商、存放位置、照片附件、备注
- **处理**：
  - 根据类别自动生成唯一编号（如 PLC-MIT-Q-004）
  - 初始状态为"在库"
  - 初始保管人为空
- **输出**：资产编号、创建成功提示
- **约束**：
  - 资产编号全局唯一
  - 必填项：类别、名称、存放位置
  - 照片附件最多5张，单张≤5MB

#### FR-1.2 资产编辑
- **描述**：修改资产基本信息
- **约束**：
  - 资产编号不可修改
  - 状态为"借出中"时仅可修改备注和附件
  - 记录修改前后差异到审计日志

#### FR-1.3 资产报废
- **描述**：将资产标记为报废
- **处理**：
  - 检查当前状态（仅"在库"状态可报废）
  - 逻辑删除（is_deleted=true）
  - 记录报废操作人和时间
- **约束**：报废后资产不可恢复，但历史记录保留

#### FR-1.4 资产查询
- **筛选条件**：
  - 资产编号（模糊匹配）
  - 资产名称（模糊匹配）
  - 类别（支持多选）
  - 状态（在库/借出中/维修中/报废）
  - 存放位置（支持多选）
  - 当前保管人
- **排序**：编号、创建时间、更新时间
- **分页**：默认每页20条

#### FR-1.5 资产详情
- **展示内容**：
  - 基本信息（编号、名称、规格、品牌等）
  - 当前状态和保管人
  - 附件照片（点击查看大图）
  - 流转记录时间线（借出→归还历史）
  - 最近5条操作日志

### 3.2 类别与位置管理

#### FR-2.1 类别管理
- **树形结构**：支持二级分类（如 电气设备 > PLC）
- **编号规则配置**：
  - 多段式规则（固定段+流水号段）
  - 示例：`[固定:PLC]-[固定:MIT]-[固定:Q]-[流水:001]`
  - 流水号支持配置起始值、位数、是否按年重置
- **操作**：新增、编辑、删除（有关联资产时禁止删除）

#### FR-2.2 位置管理
- **树形结构**：支持三级（如 一号仓库 > A区 > A01货架）
- **二维码生成**：每个位置自动生成二维码（含位置ID）
- **操作**：新增、编辑、删除（有关联资产时禁止删除）

### 3.3 审批流管理（可配置工作流引擎）

审批采用**可配置工作流引擎**：在「系统管理 → 审批流程」中以流程图方式自由设计各业务（借用 / 转让 / 归还）的审批流程，支持自由增删节点、配置审批人、会签 / 或签、条件分支；运行时支持加签、转签。详见《审批工作流设计》。

> **说明**：本系统不含定时任务与邮件提醒。审批不会超时自动通过（超时仅在列表与详情中作展示标记）；通知通过站内消息发送。

#### FR-3.1 流程设计
- **操作人**：系统管理员
- **节点类型**：发起 / 审批（单人）/ 会签（所有人通过）/ 或签（一人通过）/ 条件分支 / 抄送 / 结束
- **审批人来源**：指定人 / 角色 / 直属主管（优先取有效代审人）/ 部门负责人 / 发起人指定
- **条件分支**：按申报金额等条件决定是否经过某节点（不满足则自动跳过）
- **操作**：增删节点、调整顺序、配置审批人与类型、保存流程

#### FR-3.2 借用申请
- **发起人**：任何员工
- **必填信息**：借用资产（单选，仅"在库"可借）、借用原因、预计归还日期
- **处理**：检查资产状态 → 按"借用审批流"克隆节点生成工单实例 → 流转至首个审批节点 → 站内通知审批人

#### FR-3.3 归还申请
- **发起人**：当前保管人
- **必填信息**：归还资产、归还说明（可选）
- **处理**：按"归还审批流"生成工单，由资产管理员确认入库

#### FR-3.4 转让申请
- **发起人**：任何员工（通常为当前保管人或资产管理员）
- **场景**：资产保管责任永久移交，无时间限制、无需归还
- **必填信息**：转让资产（仅"在库"可转让）、接收人、转让原因（10-200字）
- **处理**：按"转让审批流"生成工单；跨部门转让由申请人本部门主管审批；通过后保管人与归属部门转移至接收人一方（详见《多部门预留设计》）

#### FR-3.5 审批操作（运行时）
- **通过当前节点**：填写意见 → 节点完成并推进；会签需该节点所有人通过、或签一人通过即推进；条件节点按金额判定经过或跳过
- **驳回**：填写驳回理由（必填）→ 流程终止，工单状态"已驳回"
- **加签**：在当前节点临时增加审批人（前加签 / 后加签），加签人通过后方可推进
- **转签**：将当前节点审批转交他人
- **流程完成**：所有节点通过 → 执行业务动作（借用→借出、转让→归属转移、归还→入库）

#### FR-3.6 确认入库
- **操作人**：资产管理员
- **前置条件**：归还流程审批完成
- **处理**：资产状态→"在库"，保管人清空，记录归还时间

#### FR-3.7 代审（并入工作流）
- "直属主管"类节点解析审批人时，优先取主管设置的有效代审人（出差 / 休假临时授权）
- 代审仅影响新发起的申请；已在途工单不变更审批人

### 3.4 逾期管理

> 本系统不含定时任务与邮件提醒，逾期不再自动推送，改为站内查看与手动催办。

#### FR-4.1 逾期资产查看
- **判定**：当前日期 > 预计归还日期 且 资产状态="借出中"
- **入口**：统计报表 → 逾期资产
- **内容**：资产编号、名称、保管人、应还日期、逾期天数；逾期较久的行高亮
- **催办**：管理员对单条或批量逾期资产发送**站内催办**消息（无邮件）

#### FR-4.2 到期提醒
- 在逾期资产 / 借用明细报表中以**展示标记**提示即将到期的资产（不主动推送）

### 3.5 导入导出

#### FR-5.1 Excel 导入
- **两阶段流程**：
  1. **预校验**：
     - 上传Excel文件（≤10MB）
     - 逐行校验（类别是否存在、位置是否存在、必填项是否完整）
     - 返回校验报告（有效行数、无效行数、错误列表）
  2. **确认导入**：
     - 基于校验结果sessionId提交
     - 仅导入有效行
     - 生成导入报告（成功N条、失败M条、失败原因）
- **模板下载**：提供标准Excel模板

#### FR-5.2 资产列表导出
- **格式**：Excel（.xlsx）
- **内容**：当前筛选条件下的全部资产（不分页）
- **字段**：编号、名称、类别、规格型号、状态、保管人、存放位置、创建时间

### 3.6 统计报表

#### FR-6.1 资产汇总报表
- **统计维度**：
  - 按类别统计（总数、在库数、借出数）
  - 按状态统计（在库、借出中、维修中）
  - 总价值统计（按类别汇总单价×数量）
- **导出格式**：Excel

#### FR-6.2 借用明细报表
- **筛选条件**：时间范围、类别、借用人、审批状态
- **显示字段**：工单号、资产编号、借用人、申请时间、审批人、审批时间、归还时间
- **导出格式**：Excel

#### FR-6.3 逾期资产报表
- **内容**：当前所有逾期未归还资产
- **显示字段**：资产编号、名称、保管人、应还日期、逾期天数
- **排序**：按逾期天数倒序
- **导出格式**：Excel

### 3.7 系统管理

#### FR-7.1 用户管理
- **新增用户**：工号（唯一）、姓名、角色、部门、直属主管、邮箱、手机
- **编辑用户**：修改角色、主管、联系方式
- **禁用用户**：软删除，历史数据保留
- **重置密码**：管理员可重置为默认密码（工号后6位）

#### FR-7.2 系统参数配置
- **审批超时时间**（小时，默认48）
- **到期前提醒天数**（默认3）
- **审计日志保留月数**（默认12）
- **邮件服务器配置**（SMTP地址、端口、账号、密码）
- **附件存储路径**

#### FR-7.3 审计日志查询
- **筛选条件**：操作人、操作类型、时间范围、关联资产编号
- **显示字段**：操作时间、操作人、操作类型、操作摘要、变更内容（diff）
- **排序**：按时间倒序
- **导出**：Excel

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 响应时间 | 列表查询 < 1秒，详情查询 < 500ms |
| 并发支持 | 支持50人同时在线 |
| 数据量 | 支持10万条资产记录，50万条审计日志 |
| 报表导出 | 5000条数据导出 < 10秒 |

### 4.2 可用性需求

- **系统可用性**：工作时间（8:00-18:00）可用率 ≥ 99%
- **数据备份**：每日自动备份数据库，保留最近30天
- **故障恢复**：系统故障后1小时内恢复

### 4.3 安全需求

- **身份认证**：JWT Token机制，有效期8小时，支持自动续签
- **密码策略**：初始密码为工号后6位，强制首次登录修改
- **权限控制**：基于角色的访问控制（RBAC）
- **数据传输**：内网HTTPS加密
- **操作审计**：所有写操作自动记录审计日志

### 4.4 兼容性需求

- **浏览器支持**：Chrome 90+、Edge 90+（不支持IE）
- **设备支持**：PC（1920×1080）、平板（1024×768横屏）
- **数据库**：SQL Server 2017+

### 4.5 可维护性需求

- **代码规范**：遵循SOLID原则，分层清晰
- **日志记录**：记录错误日志（含trace_id）、审计日志、性能日志
- **健康检查**：提供 `/health` 接口检测数据库、定时任务状态

### 4.6 用户体验需求

- **操作反馈**：所有操作提供明确的成功/失败提示
- **加载状态**：列表查询、报表导出等耗时操作显示加载动画
- **确认弹窗**：删除、报废等不可逆操作需二次确认
- **表单验证**：实时校验，错误提示在字段旁显示
- **空状态提示**：列表无数据时显示友好的空状态提示
- **快捷操作**：支持Enter提交表单，Esc关闭弹窗

## 5. 业务规则

### 5.1 资产编号规则

- **分类层级逐层构建**：资产分类为任意多层编码树，每层节点含一个编码段（如 PLC / MIT / Q）；完整编码 = 各层编码段自上而下拼接（PLC-MIT-Q），可在资产列表中逐层增删改查
- 资产编号 = 末层分类编码 + 流水号（如 PLC-MIT-Q-001）
- 编号全局唯一，一经生成不可修改
- 支持多段式配置（固定段+流水号段）
- 流水号并发生成采用乐观锁，失败最多重试3次
- 流水号允许空洞（事务回滚时不回退序列），不允许重复

### 5.2 状态流转规则

```
在库 ──借用审批通过──→ 借出中 ──归还确认入库──→ 在库
 │  ↑                                           │
 │  └─ 转让审批通过（状态不变，保管人→接收人）     │
 │                                              │
 └────────────── 直接报废 ──────────────────────┘

约束：
- 借出中状态不可直接报废（需先归还）
- 维修中状态可直接流转到在库或借出中（由管理员手动操作）
- 转让不改变资产状态（始终为"在库"），仅变更保管人，无时间限制、不纳入逾期管理
- 借出中的资产不可直接转让，须先归还再转让
```

### 5.3 审批规则

- 申请人不能审批自己的申请
- 审批人离职/禁用时，工单转由系统管理员处理
- 代审配置仅影响新工单，已存在的待审批单不变更审批人
- 审批不超时自动通过（无定时任务）；超时仅在列表与详情中作展示标记，需人工处理

### 5.4 权限规则

- 普通员工只能查看自己的申请记录
- 主管可查看所有下属的申请记录（含间接下属）
- 资产管理员和系统管理员可查看全部记录

### 5.5 数据校验规则

#### 资产档案字段校验
- **资产名称**：1-100字符，不允许纯空格
- **规格型号**：最大200字符
- **单价**：0-999,999,999.99，保留2位小数
- **数量**：1-99,999，正整数
- **供应商**：最大100字符
- **备注**：最大500字符

#### 审批工单字段校验
- **借用原因**：10-500字符，必填
- **预计归还日期**：必须 ≥ 当前日期，最长不超过1年
- **审批意见**：最大200字符
- **驳回理由**：10-200字符，必填

#### 用户字段校验
- **工号**：6-20位，字母或数字
- **姓名**：2-50字符，中英文
- **邮箱**：标准邮箱格式
- **手机**：11位数字（中国大陆）

### 5.6 异常场景处理

#### 审批异常
- **超时后手动审批**：超时自动通过后，手动审批按钮置灰，不可再操作
- **审批人变更**：审批人离职时，待审批单自动转给系统管理员

#### 并发冲突
- **编号生成冲突**：乐观锁失败后自动重试3次，仍失败提示用户稍后重试
- **资产状态冲突**：同时发起多个借用申请时，只有第一个成功，其他提示"资产已被借出"

#### 网络异常
- **导入中断**：预校验session有效期30分钟，超时需重新上传
- **导出失败**：生成失败时删除临时文件，不留垃圾数据
- **邮件发送失败**：写入失败日志，异步重试3次，最终失败不影响主流程
