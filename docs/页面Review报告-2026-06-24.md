# 页面 Review 报告（2026-06-24）

## 结论

本次已覆盖前端可直接访问的认证页与后台动态菜单页。后台主流程可以登录、路由可以加载，未发现稳定复现的接口 4xx/5xx 或页面级崩溃。

主要问题集中在三类：

1. **认证页仍暴露模板化/未落地功能**：验证码登录、二维码登录、忘记密码路由可直接访问，文案和交互仍是 Vben 模板状态。
2. **移动端页面不是完整适配**：首页统计卡片、报表筛选区、多列表格在 390px 宽度下可读性明显下降。
3. **暗色主题下自定义页面和 Element Plus 表格样式不一致**：页面容器大量硬编码白底，表格又保留深色单元格，视觉层级割裂。

## 关键问题

### P1：认证页暴露未落地功能和模板文案

受影响页面：

- `/auth/login`
- `/auth/code-login`
- `/auth/qrcode-login`
- `/auth/forget-password`

证据：

- `web\apps\web-ele\src\router\routes\core.ts:50` 到 `web\apps\web-ele\src\router\routes\core.ts:74` 注册了验证码登录、二维码登录、忘记密码路由。
- `web\packages\locales\src\langs\zh-CN\authentication.json:3` 到 `web\packages\locales\src\langs\zh-CN\authentication.json:7` 仍是“开箱即用的大型中后台管理系统”“工程化、高性能、跨组件库的前端模版”等模板文案。
- `web\packages\locales\src\langs\zh-CN\authentication.json:31`、`:36`、`:39` 描述了邮箱重置、扫码、手机号验证码，但当前系统没有对应真实闭环。
- `web\packages\effects\common-ui\src\ui\authentication\code-login.vue:91`、`qrcode-login.vue:70`、`forget-password.vue:87` 标题内有 emoji，和项目后台风格不一致。

影响：

- 用户直接访问这些 URL 会看到不可用或不符合本系统的登录方式。
- “忘记密码”会暗示可发送邮箱重置链接，但内网资产管理系统当前没有这个能力。
- 登录页宣传语不是资产管理业务语言，降低交付完整度。

建议：

- 若暂不支持验证码/扫码/找回密码，直接从路由表移除或重定向到 `/auth/login`。
- 登录页文案改为资产管理语境，例如“部门资产管理系统”“请输入工号和密码登录”。
- 去掉认证页标题 emoji，保持企业后台一致性。

### P1：移动端首页统计卡片被强制四列，文字竖排

受影响页面：`/home`

证据：

- `web\apps\web-ele\src\views\dashboard\workspace\index.vue:125` 在模板里写了内联样式 `grid-template-columns: repeat(4, 1fr);`。
- `web\apps\web-ele\src\views\dashboard\workspace\index.vue:474` 到 `:481` 只调整了工作区网格，没有覆盖顶部统计卡片。
- 移动截图中“资产总数 / 在库资产 / 借出资产 / 待审批”等文字被压成窄列，阅读体验很差。

影响：

- 移动端首页首屏最重要的 KPI 区域不可读。
- 内联样式优先级高，后续 CSS 响应式规则不容易覆盖。

建议：

- 删除内联 grid 样式，改成类名控制。
- 桌面 `repeat(4, minmax(0, 1fr))`，平板 `repeat(2, 1fr)`，移动 `1fr` 或横向两列。

### P1：移动端报表筛选区和宽表没有移动布局

受影响页面：

- `/report/borrow`
- `/report/summary`
- `/report/overdue`
- 同类问题也影响审批、资产、系统管理中的多列表格页面。

证据：

- `web\apps\web-ele\src\views\report\borrow\index.vue:112` 到 `:141` 使用 inline 表单，并给日期、输入框、下拉框写固定宽度。
- `web\apps\web-ele\src\views\report\borrow\index.vue:145` 到 `:171` 是 8 列报表表格，移动端没有列优先级或卡片视图。
- `web\apps\web-ele\src\views\report\summary\index.vue:76` 到 `:108` 两个统计表在移动端虽会变一列，但每张表仍是固定多列表格。

影响：

- 移动端筛选条件互相挤压，表格阅读成本高。
- 对“借用明细”“逾期资产”这种查询页，移动端只能勉强查看，不能高效操作。

建议：

- 筛选区移动端改成纵向表单或“筛选”抽屉。
- 移动端表格改成业务卡片列表，只保留主信息，次要字段折叠到详情。
- 桌面保留宽表，移动端用 `v-if`/CSS 容器查询切换结构。

### P2：大量宽表依赖桌面表格模型

受影响页面：

- `/asset/list`
- `/approval/pending`
- `/approval/mine`
- `/approval/confirm-return`
- `/report/borrow`
- `/report/overdue`
- `/admin/users`
- `/admin/roles`
- `/admin/audit`
- `/admin/departments`
- `/admin/workflows`
- `/admin/settings`

证据：

- `web\apps\web-ele\src\views\asset\list\index.vue:598` 到 `:636` 是 10 列资产表，并固定右侧操作列。
- `web\apps\web-ele\src\views\approval\pending\index.vue:240` 到 `:275` 是 7 列审批表。
- `web\apps\web-ele\src\views\approval\mine\index.vue:115` 到 `:150` 是 6 列申请表。
- `web\apps\web-ele\src\views\approval\confirm-return\index.vue:119` 到 `:149` 是 10 列确认入库表。
- `web\apps\web-ele\src\views\admin\users\index.vue:231` 到 `:268`、`admin\roles\index.vue:263` 到 `:292` 均是固定操作列宽表。

影响：

- 桌面端可用，但移动端表格信息密度过高，操作列占据过多空间。
- 低数据量时页面显得空旷；高数据量时移动端会非常难读。

建议：

- 移动端给资产、审批、报表、用户、角色页统一抽象 `ResponsiveRecordList` 或局部卡片列表。
- 桌面表格保留；移动端显示“标题 + 状态 + 关键字段 + 操作菜单”。

### P2：暗色主题下自定义样式和组件主题割裂

受影响页面：几乎所有业务页。

证据：

- 多处页面容器硬编码白底和浅色边框，例如 `web\apps\web-ele\src\views\report\summary\index.vue:165` 到 `:170`。
- 表格区域另有大量硬编码浅色表头和边框，例如 `web\apps\web-ele\src\views\asset\list\index.vue:1057` 到 `:1092`。
- 截图中页面大容器是白底，表格单元格却接近黑色，卡片/表格层级不统一。

影响：

- 暗色主题看起来像局部“半成品主题适配”。
- 后续新页面容易继续复制硬编码颜色，主题债务扩散。

建议：

- 自定义页面样式统一使用 Element Plus / Vben 主题变量，不直接写 `#ffffff`、`#f8f9fa`、`#1e293b`。
- 建议先抽一套业务页 token：页面背景、卡片背景、表格头、边框、弱文本、主文本。

### P2：全局头部图标按钮缺少可访问名称

受影响页面：所有后台页面。

证据：

- 自动巡检每个后台页都检测到 7 到 8 个图标按钮缺少 `aria-label` 或 `title`。
- 主要集中在顶部工具栏：搜索、布局、语言、主题、全屏、通知等。

影响：

- 屏幕阅读器无法说明按钮用途。
- 键盘用户聚焦时语义不清。

建议：

- 给头部工具栏图标按钮补 `aria-label`/`title`，优先处理搜索、主题、全屏、通知。

### P2：资产列表“进入”是整卡点击，不是真正按钮/链接

受影响页面：`/asset/list`

证据：

- `web\apps\web-ele\src\views\asset\list\index.vue:462` 到 `:469` 将 `article` 作为点击入口。
- `web\apps\web-ele\src\views\asset\list\index.vue:481` 只是视觉文本“进入 →”。

影响：

- 可点击区域不够明确，自动化与键盘用户都不容易发现。
- `article tabindex="0"` 可以回车进入，但语义仍不如真实按钮或链接清晰。

建议：

- 把“进入”改为真实 `ElButton link` 或 `<button>`，保留整卡点击也可以，但需要明确焦点样式和可访问名称。

## 复测说明

初次自动巡检中 `/approval/mine`、`/approval/confirm-return`、移动端 `/report/summary` 出现过 Vite 模块请求 `ERR_INSUFFICIENT_RESOURCES`，页面停在全局 loading。随后使用全新浏览器上下文逐页复测，三页均正常渲染，且无控制台错误。因此本报告不把它定为业务缺陷，只记录为开发服务器快速切页下的资源波动。

复测结果：

- `/approval/mine`：桌面 `bodyLength=148`，移动 `bodyLength=82`，loading 为 `0`。
- `/approval/confirm-return`：桌面 `bodyLength=113`，移动 `bodyLength=46`，loading 为 `0`。
- `/report/summary`：桌面 `bodyLength=226`，移动 `bodyLength=151`，loading 为 `0`。

## 建议修复顺序

1. 先处理认证页：移除未实现认证路由或重定向，并替换模板文案。
2. 修首页移动端 KPI 卡片：删除内联四列样式，补响应式 grid。
3. 统一移动端表格策略：审批、报表、系统管理页先做一套卡片化列表模式。
4. 收敛暗色主题 token：禁止业务页继续硬编码浅色背景/边框。
5. 补全顶部工具栏图标按钮 `aria-label`。
