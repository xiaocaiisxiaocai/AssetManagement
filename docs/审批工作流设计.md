# 审批工作流引擎设计（用友式自由流程）

**文档版本**：v1.0
**编写日期**：2026-06-11
**性质**：功能设计 / ADR
**关联文档**：需求文档、设计概要、架构设计文档、多部门预留设计

---

## 1. 背景与目标

当前审批是**固定流程**：申请 → 直属主管/代审人审批 → 通过/驳回，审批人由"直属主管"规则写死，并依赖定时任务（超时自动审批、逾期提醒）与邮件通知。

业务方要求审批"像用友那样自由设计"——可自由加节点、加签、会签/或签、按条件分支。本设计把固定审批升级为**可配置的工作流引擎 + 流程图式设计器**，并**移除定时任务与邮件提醒**。

## 2. 范围边界（已与业务方确认）

**原型实现**：流程图式可视化设计器、节点类型（发起/审批/会签/或签/条件/抄送/结束）、审批人来源（指定人/角色/直属主管/部门负责人/发起人自选）、借用/转让/归还各一套可编辑流程、运行时按定义流转（时间线）、审批操作（通过/驳回/加签/转签）、会签全通过推进、或签一人推进、条件分支按金额走不同路径。

**列入演进（原型不做）**：并行网关汇聚、子流程、流程版本管理、表单设计器、流程模拟器。

## 3. 数据模型

### 3.1 流程定义 `workflows`
```
{
  id, name, bizType(borrow/transfer/return),
  nodes: [
    { id, name, type, approverType, approver, signers[], condition }
  ]
}
```
- **节点类型 type**：`start` 发起 / `approval` 单人审批 / `countersign` 会签（所有人通过）/ `orsign` 或签（一人通过）/ `condition` 条件分支 / `cc` 抄送 / `end` 结束
- **审批人来源 approverType**：`user` 指定人 / `role` 角色 / `supervisor` 直属主管（解析时优先有效代审人）/ `deptManager` 部门负责人 / `applicantPick` 发起人指定
- **condition**：条件节点的进入条件（如 `金额 > 5000`），不满足则跳过该节点

### 3.2 审批实例（工单升级）
在原 `approvals` 工单上增加：
```
workflowId, amount, currentNodeIndex,
nodes: [
  { name, type, status(done/current/pending/skipped), approver, opinion, time,
    signers[], signStates{}, addedSigners[] }
]
```

## 4. 流程设计器（系统管理 → 审批流程）

- 流程图式纵向：节点卡片 + 连线箭头 + 条件分支双路（是/否）
- 顶部切换流程（借用/转让/归还）
- `[+ 添加节点]`：选节点类型插入；点击节点弹出配置（节点名、类型、审批人来源、会签/或签成员、条件表达式）
- 节点操作：上移 / 下移 / 删除（发起与结束节点固定）
- 保存流程（原型写入内存 `mockData.workflows`）

## 5. 运行时

- **发起**：按 bizType 匹配流程模板，克隆 nodes 生成实例，首个审批节点置 `current`
- **时间线**：审批详情按节点展示——已办 ✓（审批人+意见+时间）、当前 ⏳、待办 ○、跳过 ⊘
- **审批操作**：
  - **通过**：`approval` → 节点 done，推进；`countersign` 会签 → 记录该人通过，`signStates` 全通过才 done 推进；`orsign` 或签 → 一人通过即 done 推进；`condition` → 按 `amount` 判定，满足则审批、不满足置 `skipped` 直接推进
  - **驳回**：工单 `rejected`，流程终止
  - **加签**：当前节点 `addedSigners` 增加审批人（前加签=需先于原审批人、后加签=原审批人之后），加签人全部通过才算节点通过
  - **转签**：当前节点 `approver` 改为他人
  - **抄送 cc**：自动通过并记录知会（站内，无邮件）
- **完成**：所有节点 done → 工单 `approved`，执行业务动作（借出 / 转让归属转移 / 归还入库）

## 6. 移除定时任务与邮件

- 删除：超时自动审批、逾期提醒定时任务；全部邮件通知；系统参数中的 SMTP、审批超时小时数、逾期催还间隔等配置项
- **超时**：保留"审批截止"展示与列表"已超时"标记，但不自动处理（纯人工）
- **代审设置**：并入引擎——`supervisor` 类节点解析审批人时优先取有效代审人；不再作为独立定时逻辑
- **逾期资产报表**：保留查看，催还由"邮件"改为"站内手动催办"

## 7. 演进

并行网关与汇聚、子流程、流程版本管理与灰度、在线表单设计器、流程模拟与压测。届时审批实例需引入有向图（而非线性 nodes 数组）与网关节点。

## 8. 测试
