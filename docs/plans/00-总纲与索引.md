## 全局约定

- **统一响应**：`ApiResult<T> { int Code; string Message; T? Data; }`，`Code=0` 表示成功。**HTTP 状态码约定**：业务成功与业务错误均返回 **HTTP 200** + `Code`（前端统一用 `code` 判断）；但**认证失败返回 401、授权失败返回 403**（标准状态码，不包成 200），以便 vben 请求拦截器触发重新登录、Swagger 与浏览器正确识别。
- **分页**：`PagedResult<T> { IReadOnlyList<T> Items; int Total; int Page; int PageSize; }`。
- **异常**：全局异常中间件捕获 → `ApiResult`（业务异常 `BizException(code,msg)` → 200 + code；未捕获异常 → 200 + Code=500）。**认证/授权异常不经此包装**——由 JWT 中间件与授权中间件直接产出 401/403。
- **认证/鉴权**：JWT Bearer；自定义 `[HasPermission("asset:create")]` 策略，权限码 `模块:动作`；未登录 → 401，已登录但无权限 → 403。
- **命名**：C# 类型/方法 PascalCase；EF 实体类单数（`Asset`），表名由 `ToTable` 指定复数 snake_case（`assets`）；DTO 后缀 `Dto`/`Request`/`Response`。
- **时间**：实体存 UTC（`DateTime` UTC）；前端展示本地时区。
- **版本控制**：当前目录尚未初始化 Git（见 CLAUDE.md / AGENTS.md）。**M0 第一个任务先 `git init`**（见 `M0-脚手架.md` Task 0），之后每个任务一次提交，Conventional Commits（`feat:`/`fix:`/`test:`/`chore:`/`docs:`）。若团队偏好集中提交，可把每任务 commit 视为"阶段检查点"。
- **测试**：领域规则（编号拼接、工作流流转判定）必做 TDD；关键 API 写集成测试（`WebApplicationFactory`，用内存/临时 SQLite）。
- **不引入计划外依赖**；新增 NuGet 包在任务中显式 `dotnet add package`。

## 前端页面实现标准步骤（M2–M6 前端任务统一按此拆解）

每个业务页面按以下 6 个小步实现（每步 2–5 分钟、可单独提交），替代"参照原型一笔带过"：

1. **路由与菜单**：在 vben 路由表注册该页路由；菜单由后端 `menu/routes` 驱动，确认对应权限码。
2. **API client**：在 `web/src/api/<模块>.ts` 新建请求函数（对应后端接口，定义 TS 请求/响应类型）。
3. **页面骨架**：页面组件 + 面包屑 + 页头 + 空容器，路由可达、能渲染。
4. **数据展示**：表格 / 树 / 卡片绑定 API（分页 / 筛选 / 树形按需），加载与空态。
5. **表单与操作**：新增 / 编辑弹窗、删除二次确认、业务操作按钮，对接写接口并刷新。
6. **权限点与冒烟**：按钮按权限码显隐（vben 权限指令）；对照 `prototype/` 对应页交互与文案手动冒烟一遍。

各里程碑前端任务只需列出"该页用到的具体组件 / 接口清单"，实现时套用以上 6 步。
