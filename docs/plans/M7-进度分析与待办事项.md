# 部门资产管理系统 — 进度分析与待办事项

**分析日期**：2026-06-17 | **性质**：项目完备性评估与差距分析报告

## 1. 总体完成度概览

本系统已完成了约 **85%** 的全栈开发工作。后端采用 **ASP.NET Core 8 + EF Core + SQLite**，前端采用 **Vue 3 + Element Plus (web-ele)**。四大核心模块（资产管理、工作流配置与审批流转、报表统计、RBAC/基础数据系统）已基本打通，并接入了操作审计和系统参数配置。

然而，通过对前/后端源码的深度地毯式审查，发现目前存在部分**功能性缺失**、**前后端接口未对齐 Bug** 以及**数据层面的设计差距**。

## 2. 模块级详细完成度与接口映射

### 2.1 认证与 RBAC 模块 (M1)
* **后端实现**：`AuthController.cs`、`UserController.cs`、`RoleController.cs`、`RbacMenuController.cs`、`PermissionController.cs` 均已完整实现。支持 JWT 颁发与验证，支持基于权限码的 `[HasPermission]` 动态策略鉴权。
* **前端对接**：`views/_core/authentication/login.vue` 已打通并进行了定制，屏蔽了无关登录方式；`views/admin/users/` 与 `views/admin/roles/` 页面功能完全打通，实现了用户、角色、关联菜单与权限码的管理。
* **完成度**：🟢 **98%** (已完全可用)

### 2.2 基础数据管理 (M2)
* **后端实现**：`DepartmentController.cs`、`LocationController.cs`、`AssetCategoryController.cs`、`SettingController.cs`。实现了树形部门、树形存放位置、树形资产分类的 CRUD。
* **前端对接**：`views/admin/departments/index.vue`、`views/asset/locations/index.vue`、`views/asset/categories/index.vue`、`views/admin/settings/index.vue`，均与后端接口完全连通。
* **完成度**：🟢 **95%** (基础功能全绿，其中二维码生成及位置打印仅留有前端展示，暂无物理打印联调)

### 2.3 资产管理模块 (M3)
* **后端实现**：`AssetController.cs`、`AssetImportController.cs`。实现了资产的分页查询、CRUD、自动流水号拼接生成、Excel 导入预校验及二阶段确认导入。
* **前端对接**：`views/asset/list/index.vue` 提供层叠分类和纯列表双模式切换，集成了资产新增/修改表单、导入/导出和流程发起。
* **完成度**：🟡 **75%** (存在核心功能缺失，详见第 3 节)

### 2.4 审批工作流模块 (M4)
* **后端实现**：`WorkflowController.cs`、`ApprovalController.cs`。基于纯函数引擎逻辑运行，支持会签/或签/分支条件/加签/转签。`BizEffectApplier` 用于在审批流结束后执行副作用（如借出状态变更、保管人变更、转让时部门变更）。
* **前端对接**：
  * `views/admin/workflows/index.vue` 基于 `LogicFlow` 实现了可视化的流程图设计器，前后端数据完全连通。
  * `views/approval/pending/index.vue` (待我审批) 支持审批、驳回、加签、转签。
  * `views/approval/mine/index.vue` (我的发起) 支持查看历史发起详情。
* **完成度**：🟡 **80%** (存在严重的入库接口对齐 Bug，详见第 3 节)

### 2.5 报表与审计模块 (M5)
* **后端实现**：`ReportController.cs`、`AuditLogController.cs`。实现了汇总报表、借用报表、逾期报表以及审计日志的数据查询与 Excel 二进制文件流导出。
* **前端对接**：`views/report/summary/`、`views/report/borrow/`、`views/report/overdue/`、`views/admin/audit/` 前后端打通。
* **完成度**：🟢 **95%**

## 3. 核心缺失功能与 Bug 清单 (待办事项)

### ✅ 待办 1（已完成 2026-06-17）：确认入库接口对齐
* **现状（修复前）**：前端 `approval/confirm-return/index.vue`（确认归还入库页面）调用的是 `getMineApprovalsApi()` 接口（对应后端的 `/api/approvals/mine`）。该接口仅获取**当前登录用户自己发起**的工单。
* **后果**：资产管理员（Warehouse 角色）无法在此页面看到“别人发起的且已被主管通过”的待入库工单，导致**归还入库确认功能完全瘫痪**。
* **修复实现**：
  1. 后端 `IWorkflowService.PendingReturnsAsync()` + `WorkflowService` 实现：返回全局 `Status=="approved" && BizType=="borrow" && ConfirmedAt==null` 的借用流程单（不限发起人）。
  2. `ApprovalController` 新增 `GET /api/approvals/pending-return`，`[HasPermission("asset:edit")]`（与 `confirm-return` 动作权限一致）。
  3. 前端 `api/workflow.ts` 新增 `getPendingReturnsApi`；`confirm-return/index.vue` 改为调用此接口。
  4. 附带：给 `ApprovalFlow` 前端类型补 `confirmedAt` 字段、修正该页 `total > pageSize` 分页 bug。
  5. 新增集成测试 `ApprovalApiTests.Pending_return_lists_approved_unconfirmed_borrow_flow`（RED→GREEN，全套 47 测试通过）。

### ✅ 待办 2（已完成 2026-06-17）：FR-1.5 资产详情页及流转时间线
* **现状（修复前）**：需求和原型中要求有”资产详情”，可展示：(1) 基本信息；(2) 状态与保管人；(3) 附件照片大图；(4) **资产流转历史时间线**（借出->归还历史）；(5) **最近 5 条操作日志**。目前 `list/index.vue` 没有提供详情弹窗或页面入口，也没有实现对应的前后端调用。
* **修复实现**：
  1. 后端 `AssetService.GetDetailAsync()`: 聚合查询资产基本信息 + `ApprovalFlows`（按时间倒序）+ `AuditLogs`（TargetType==Asset 的最近5条）。
  2. `AssetController` 新增 `GET /api/assets/{id}/detail`，`[HasPermission(“asset:view”)]`。
  3. 前端 `api/asset.ts` 新增 `getAssetDetailApi` + `AssetDetail/AssetFlow/AssetAuditLog` 类型定义。
  4. `list/index.vue` 详情弹窗：基本信息网格 + 照片大图预览 + `ElTimeline` 流转时间线（借用/归还/转让/报废，带状态标签） + 操作日志表格。列表/层级视图操作栏均加”详情”按钮。
  5. 新增集成测试 `Asset_detail_returns_flows_and_recent_logs`（创建资产 → 更新产生审计日志 → 发起借用产生流转单 → 验证 detail 接口返回完整数据，全套 51 测试通过）。

### ✅ 待办 3（已完成 2026-06-17）：FR-1.1 & FR-1.5 照片附件
* **现状（修复前）**：
  - 后端：`Asset` 实体中没有定义任何照片/图片文件路径相关的字段，数据库中无法保存资产照片。
  - 前端：资产新增/编辑表单中没有文件上传组件。
* **修复实现**：
  1. 后端 `Asset` 实体加 `string? ImageUrls`（逗号分隔，≤5 张）+ `AssetConfiguration` 长度限制 + EF 迁移 `AddAssetImageUrls`；`AssetDto/Create/UpdateAssetRequest` 加 `Images`,`AssetService` 增删改查映射并校验张数。
  2. 新增通用文件存储 `IFileStorageService`/`FileStorageService`(磁盘,可配置 `Attachment:Path`,校验 jpg/png/gif/webp 与 ≤5MB,GUID 命名防穿越) + `FileController`:`POST /api/files/upload`[asset:edit]、`GET /api/files/{name}`[匿名]。
  3. 前端 `asset.ts` 加 `uploadAssetImageApi` 与 `images` 字段；`list/index.vue` 资产表单接入 `ElUpload`(picture-card、最多 5 张、单张 ≤5MB、编辑回填、提交带 URL)。
  4. 测试:`FileApiTests`(上传/回显/拒绝非图) + `AssetApiTests.Create_asset_persists_image_urls`(全套 50 测试通过),并端到端验证上传→匿名回显→建带图资产。

### 🚨 待办 4：多部门数据权限隔离未实现
* **现状**：虽然预留设计中提到“部门管理员限管理本部门资产，查看范围限本部门”，但后端 `AssetService.cs` 并没有根据当前登录用户的部门 ID 对资产查询列表做强制数据过滤。
* **修复方案**：
  - 在后端资产查询（如 `QueryAsync`）中获取当前登录用户的角色。如果是“部门管理员 (dept_admin)”，强制在 EF 查询条件中附加上 `x.DepartmentId == currentUser.DepartmentId` 的数据隔离条件。

### 🚨 待办 5：资产层级空壳文件清理
* **现状**：`web/apps/web-ele/src/views/asset/hierarchy/index.vue` 物理文件存在但仅为展示文本的空壳。其实际功能（按分类层级展示资产）已在 `views/asset/list/index.vue` 的 `activeView === 'hierarchy'` 条件视图中高水平实现。
* **处理方案**：移除该物理空壳文件，或者将其路由从系统菜单中剔除（在后端 `DbSeeder` 的菜单初始化数据中只指向 `list/index.vue` 的路由）。
