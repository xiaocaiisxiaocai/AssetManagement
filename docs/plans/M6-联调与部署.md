# M6 联调与部署 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development 或 superpowers:executing-plans。**前置：M0–M5 全部完成。**

**Goal:** 完成前后端全链路联调，产出内网单机部署方案（后端 Kestrel/IIS + 前端静态托管 + SQLite），及初始化、备份、运维说明。

**Architecture:** 后端 `dotnet publish` 自包含发布；前端 `pnpm build` 产出静态文件，由后端 wwwroot 或 IIS/Nginx 托管（同源避免跨域）；SQLite 单文件 + 定期文件级备份。

**Tech Stack:** dotnet publish, pnpm build, Kestrel/IIS（Windows Server 内网）。

- `backend/src/AssetManagement.Api/wwwroot/`（托管前端产物，方案一）

- 根：`README.md`（更新运行与部署说明）

### Task 2: 后端发布

- [ ] **Step 1**：生产配置 `appsettings.Production.json`（连接串指向部署目录 db、Jwt:Key 用强随机、关闭详细错误）。
- [ ] **Step 2**：发布
```bash
dotnet publish backend/src/AssetManagement.Api -c Release -o deploy/api --self-contained false
```
- [ ] **Step 3**：内网托管二选一——(a) Kestrel：`dotnet AssetManagement.Api.dll`（配 Windows 服务/NSSM 守护，监听内网 IP:端口）；(b) IIS：装 ASP.NET Core Hosting Bundle，站点指向发布目录，应用池"无托管代码"。
- [ ] **Step 4**：启动后 `/swagger`、`/api/health` 内网可访问。
- [ ] **Step 5: Commit** `chore: backend production publish config and host guide`

---

### Task 3: 前端构建与托管

- [ ] **Step 1**：设置前端生产 API 基址（同源相对路径 `/api` 或内网后端地址），构建：
```bash

```
- [ ] **Step 2**：托管二选一——(a) 拷贝 `web/dist` 到后端 `wwwroot`，`Program.cs` 加 `app.UseDefaultFiles(); app.UseStaticFiles();` + SPA 回退到 `index.html`；(b) IIS/Nginx 单独站点托管 `dist`，`/api` 反代到后端。vue-router history 模式需配回退规则。
- [ ] **Step 3**：浏览器访问站点根，登录走通。
- [ ] **Step 4: Commit** `chore: frontend production build and static hosting`

### Task 4: SQLite 初始化与备份

- [ ] **Step 1**：首次部署：后端启动自动 `Migrate()` 建库 + 种子（admin/角色/权限/菜单/流程）。确认 `assetmgmt.db` 生成在可写目录。
- [ ] **Step 2**：备份脚本 `deploy/backup.ps1`：定期复制 db 文件到内网 NAS（用 Windows 计划任务调度——这是运维层定时复制文件，非应用内定时任务）。
```powershell
Copy-Item "C:\app\assetmgmt.db" "\\nas\backup\assetmgmt_$(Get-Date -Format yyyyMMdd_HHmm).db"
```
