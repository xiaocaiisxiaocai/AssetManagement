# M3 资产管理 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development 或 superpowers:executing-plans。**前置：M2 完成。**

**Goal:** 实现资产档案的增删改查、按末层分类编码自动生成资产编号、列表筛选分页（按部门/分类/状态）、Excel 两阶段导入与导出。

**Architecture:** Asset 关联分类(末层)/部门/位置/保管人；编号生成 = 末层分类 Code + 递增流水号（领域服务，并发用唯一索引兜底）。导入用 EPPlus 先预校验再确认。

**Tech Stack:** EF Core, EPPlus（`dotnet add package EPPlus`，注意 License 设置 NonCommercial/自有授权）。

public static class AssetNoGenerator

        => $"{categoryCode}-{(existingCount + 1):D3}";
}

### Task 3: 资产 CRUD + 列表筛选分页

**Files:** `AssetService`, `AssetController`, DTO, `tests/.../AssetApiTests.cs`

- [ ] **Step 1: DTO**

```csharp
public record AssetQuery { public string? AssetNo; public string? Name; public int? CategoryId; public int? DepartmentId; public AssetStatus? Status; public int Page=1; public int PageSize=20; }
public record CreateAssetRequest { public string Name=""; public int CategoryId; public int? DepartmentId; public int? LocationId; public string? Model; public decimal Price; }
```

- [ ] **Step 2: 服务实现**

`CreateAsync`：查末层分类 Code → `existingCount = Assets.Count(a=>a.CategoryId==catId)` → `AssetNo = AssetNoGenerator.Next(code, existingCount)` → 插入（唯一索引兜底并发冲突，冲突则重试一次取最新 count）。
`QueryAsync`：按 `AssetQuery` 组装 EF 表达式（含 `DepartmentId` 时用 M2 部门子树 id 集合做"上卷"过滤——查部门含子部门），返回 `PagedResult<AssetDto>`。

- [ ] **Step 3: Controller**

`GET /api/assets`（`[HasPermission("asset:view")]`，分页筛选）、`GET /api/assets/{id}`、`POST /api/assets`（`asset:create`）、`PUT /api/assets/{id}`（`asset:edit`）、`DELETE`（`asset:delete`，借出中禁止删 → `BizException`）。

### Task 4: Excel 导入导出（EPPlus）

**Files:** `AssetImportService`, `AssetImportController`, `AssetController`（导出）

- [ ] **Step 1: 安装 EPPlus**

```bash
dotnet add backend/src/AssetManagement.Infrastructure package EPPlus
```
设置许可证上下文（启动时 `ExcelPackage.LicenseContext = LicenseContext.NonCommercial;` 或按授权）。

- [ ] **Step 2: 模板与导出**

`GET /api/assets/import/template` → 返回带表头的 xlsx（名称/分类编码/型号/品牌/单价）。
`GET /api/assets/export?...同筛选` → 按当前筛选导出 xlsx（FileResult）。

- [ ] **Step 3: 两阶段导入**

`POST /api/assets/import/validate`（上传文件）→ 解析每行，校验（分类编码存在、单价数字、必填）→ 返回 `List<ImportPreviewRow{ row, data, isValid, error }>`，不写库。
`POST /api/assets/import/confirm`（上传同文件或传校验通过的数据）→ 仅写入合法行，逐行生成 AssetNo，返回成功/失败计数。

- [ ] **Step 4: 集成测试**：构造内存 xlsx（2 行，1 行分类编码不存在）→ validate 返回 1 valid + 1 invalid；confirm 后资产数 +1。→ PASS。

- [ ] **Step 5: Commit** `feat: add asset excel import (two-phase) and export`
