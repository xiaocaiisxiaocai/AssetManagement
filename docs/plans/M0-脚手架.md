# M0 脚手架 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 建立后端四层解决方案（.NET 8 + EF Core + SQLite + Swagger）与公共基础设施，并把 purest-admin vben 前端本地跑通，作为后续所有里程碑的地基。

**Architecture:** 经典分层（Domain/Application/Infrastructure/Api）+ 独立测试工程。所有请求经统一响应包装与全局异常中间件。EF Core Code First + SQLite 单文件，启动时自动迁移。

**Tech Stack:** .NET 8, ASP.NET Core Web API, EF Core 8 (Sqlite), Swashbuckle, xUnit, FluentAssertions；前端 purest-admin vben（pnpm）。

- [ ] **Step 1: 实现中间件**

```csharp
using System.Text.Json;
using AssetManagement.Application.Common;
namespace AssetManagement.Api.Middleware;

public class ExceptionMiddleware
{
    private readonly RequestDelegate _next;
    private readonly ILogger<ExceptionMiddleware> _log;
    public ExceptionMiddleware(RequestDelegate next, ILogger<ExceptionMiddleware> log) { _next = next; _log = log; }

    public async Task Invoke(HttpContext ctx)
    {
        try { await _next(ctx); }
        catch (BizException ex)
        {
            await Write(ctx, ex.Code, ex.Message);
        }
        catch (Exception ex)
        {
            _log.LogError(ex, "Unhandled");
            await Write(ctx, 500, "服务器内部错误");
        }
    }

    private static async Task Write(HttpContext ctx, int code, string msg)
    {
        ctx.Response.StatusCode = 200;
        ctx.Response.ContentType = "application/json; charset=utf-8";
        var body = JsonSerializer.Serialize(new ApiResult<object?> { Code = code, Message = msg });
        await ctx.Response.WriteAsync(body);
    }
}
```

- [ ] **Step 2: 注册中间件（Program.cs，放在管道前部）**

```csharp
app.UseMiddleware<AssetManagement.Api.Middleware.ExceptionMiddleware>();
```

> 注：认证（401）/授权（403）由后续 `app.UseAuthentication()` / `app.UseAuthorization()` 直接 short-circuit 产出，**不抛异常、不经本中间件**；本中间件只包装 `BizException`（200+code）与未捕获异常（200+Code=500）。因此"业务用 200+code、鉴权用 401/403"两者并存。
