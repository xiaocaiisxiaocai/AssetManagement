## 二、发现的问题

### 🔴 P1 - 需要立即修复

**【问题1】登录后跳转404**

- **现象**: 用户登录成功后看到"哎呀！未找到页面"
- **原因**: 登录成功后跳转到 `/home` 路由，但该路由未定义
- **影响**: 所有用户登录体验受影响
- **建议修复**:
  ```javascript
  // 方案1: 添加路由重定向 (推荐)
  { path: '/home', redirect: '/asset/list' }
  
  // 方案2: 修改登录跳转逻辑
  // 在 apps/web-ele/src/router 或 store/auth.ts 中修改
  ```

### 🟡 P2 - 用户体验优化

**【问题2】资产页面弹窗遮挡**
- **现象**: 存在未关闭的"新增分类"对话框
- **建议**: 页面切换时自动关闭所有弹窗

**【问题3】按钮选择器不匹配**
- **现象**: 自动化测试无法识别"新增"按钮
- **需要**: 确认实际按钮文本

**【问题4】登出功能识别失败**
- **现象**: 自动化测试无法找到登出按钮
- **建议**: 统一登出元素的class或id
