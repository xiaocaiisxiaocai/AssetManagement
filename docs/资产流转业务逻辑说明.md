# 资产流转业务逻辑说明

**更新日期**: 2026-06-24  
**版本**: v1.0

---

## 📋 业务类型定义

系统支持三种资产流转业务类型：

| 业务类型 | BizType | 说明 | 工作流 |
|---------|---------|------|--------|
| **借用** | `borrow` | 临时借用资产，需归还 | 直属主管审批 |
| **转让** | `transfer` | 永久转移资产所有权 | 直属主管审批 → 接收部门负责人审批 |
| **归还** | `return` | 借用到期后归还资产 | 直属主管审批（预留） |

---

## 🔄 资产状态说明

### 状态枚举

```csharp
public enum AssetStatus
{
    Available = 0,    // 在库（可借用）
    Borrowed = 1,     // 借出（已借用）
    Maintenance = 2,  // 维修中
    Scrapped = 3      // 报废
}
```

### 状态流转规则

1. **借用流程**：
   - 初始状态：`Available` (在库)
   - 审批通过后：`Borrowed` (借出)
   - 确认入库后：`Available` (在库)
   - **部门内借用 vs 跨部门借用**：无论部门内还是跨部门，只要借出就是 `Borrowed` 状态

2. **转让流程**：
   - 初始状态：`Available` (在库)
   - 审批通过后：
     - 资产的 `DepartmentId` 更改为接收部门
     - 状态恢复为 `Available` (在库)
     - 所有权永久转移

3. **归还流程**：
   - 当前实现：借用审批通过后，由资产管理员在"待确认入库"页面确认归还
   - 预留设计：借用人主动发起归还审批流程（工作流已定义）

---

## 📊 借用与转让的区别

| 对比项 | 借用 (Borrow) | 转让 (Transfer) |
|-------|--------------|----------------|
| **性质** | 临时使用 | 永久转移 |
| **归还** | 需要归还 | 不需要归还 |
| **所有权** | 不变（仍属原部门） | 转移（属于接收部门） |
| **资产状态** | `Borrowed` | `Available` |
| **DepartmentId** | 不变 | 更改为接收部门 |
| **审批流程** | 直属主管审批 | 直属主管 + 接收部门负责人双重审批 |
| **到期管理** | 有到期日期，逾期提醒 | 无到期日期 |
| **统计报表** | 计入"借用明细"和"逾期报表" | 不计入借用统计 |

## 🎯 前端操作入口

### 资产列表页 (`/asset/list`)

操作按钮（针对单个资产）：
- **详情** - 查看资产详细信息和流转历史
- **编辑** - 修改资产基本信息（需 `asset:update` 权限）
- **借用** - 发起借用申请（需 `asset:borrow` 权限）
- **转让** - 发起转让申请（需 `asset:transfer` 权限）
- **删除** - 删除资产（需 `asset:delete` 权限）

### 审批流程页

- **待我审批** (`/approval/pending`) - 审批借用/转让申请
- **我的申请** (`/approval/mine`) - 查看我发起的借用/转让申请
- **待确认入库** (`/approval/confirm-return`) - 确认借用资产已归还入库

## 📈 统计与报表

### 工作台统计卡片

1. **资产总数** - 所有状态的资产总和
2. **在库资产** - 状态为 `Available` 的资产数量
3. **借出资产** - 状态为 `Borrowed` 的资产数量
4. **逾期资产** - 借用超过到期日期未归还的资产数量

**注意**：
- 转让的资产不计入"借出资产"（因为状态是 `Available`）
- 部门内借用和跨部门借用都计入"借出资产"

### 报表模块

#### 1. 资产汇总报表 (`/report/summary`)
- 按部门汇总：各部门的资产总数、在库数、借出数
- 按分类汇总：各分类的资产总数、在库数、借出数

#### 2. 借用明细报表 (`/report/borrow`)
- 列出所有 `BizType = 'borrow'` 且 `Status = 'approved'` 的审批流
- 包含：借用人、资产信息、借用日期、到期日期
- 支持时间范围筛选和导出 Excel

#### 3. 逾期报表 (`/report/overdue`)
- 列出所有借用超过到期日期且未确认入库的资产
- 逾期天数计算：当前日期 - 到期日期
- 严重逾期标记：逾期天数 > 7 天

## 🔄 完整流转示例

### 示例1：部门内借用

1. 员工 A（IT部门）借用本部门的笔记本电脑
2. 直属主管审批通过
3. 资产状态：`Available` → `Borrowed`
4. 资产所属部门：不变（IT部门）
5. 到期日：2026-07-01
6. 统计：计入"借出资产"
7. 到期后：IT部门资产管理员确认入库，状态恢复 `Available`

### 示例2：跨部门借用

1. 员工 B（HR部门）借用 IT 部门的投影仪
2. 直属主管审批通过
3. 资产状态：`Available` → `Borrowed`
4. 资产所属部门：不变（IT部门）
5. 到期日：2026-06-30
6. 统计：计入"借出资产"和 IT 部门的"借出数"
7. 到期后：IT部门资产管理员确认入库，状态恢复 `Available`

### 示例3：跨部门转让

1. IT部门将一台旧电脑转让给HR部门的员工 C
2. IT部门主管审批通过
3. HR部门负责人审批通过
4. 资产所属部门：IT部门 → HR部门
5. 资产状态：保持 `Available`（或从 `Available` 恢复为 `Available`）
6. 所有权永久转移，不需要归还
7. 统计：不计入"借出资产"，在 HR 部门的"在库资产"中显示

## ⚙️ 后端实现要点

### 工作流引擎 (BPMN 2.0)

**借用流程** (`borrow`):
```xml
StartEvent → UserTask(supervisor) → EndEvent
```

**转让流程** (`transfer`):
```xml
StartEvent → UserTask(supervisor) → UserTask(deptManager) → EndEvent
```

**归还流程** (`return` - 预留):
```xml
StartEvent → UserTask(supervisor) → EndEvent
```

### 业务副作用 (SideEffect)

当审批流程完成时（`Status = 'approved'`），自动触发：

**借用审批通过**:
```csharp
asset.Status = AssetStatus.Borrowed;
asset.CustodianId = flow.TransfereeId;  // 设置保管人
// DepartmentId 不变
```

**转让审批通过**:
```csharp
asset.Status = AssetStatus.Available;
asset.DepartmentId = flow.TransfereeDept;  // 更改所属部门
asset.CustodianId = flow.TransfereeId;     // 设置新保管人
```

**归还审批通过**:
```csharp
// 当前实现：不自动变更，等待手动确认入库
// 预留实现：自动恢复 asset.Status = AssetStatus.Available
```

### 确认入库接口

**路由**: `POST /api/approvals/{flowId}/confirm-return`

**功能**: 资产管理员确认借用资产已实际归还

**副作用**:
```csharp
asset.Status = AssetStatus.Available;
asset.CustodianId = null;  // 清除保管人
flow.ConfirmedAt = DateTime.Now;  // 标记已确认
```

## 🚀 未来扩展建议

### 1. 部门内借用简化流程
如果希望区分部门内和跨部门借用，可以：
- 添加新状态 `InUse`（使用中，部门内借用）
- 部门内借用：状态变为 `InUse`，审批流程简化（仅需直属主管）
- 跨部门借用：状态变为 `Borrowed`，审批流程加强（需要资产所属部门主管审批）

### 2. 主动归还流程
激活 `return` 工作流：
- 借用人在"我的申请"页面点击"申请归还"
- 发起 `BizType = 'return'` 的审批流程
- 审批通过后自动恢复 `Available` 状态

### 3. 自动到期提醒
- 定时任务：每天检查即将到期（3天内）的借用申请
- 发送通知给借用人和资产管理员
- 严重逾期（>7天）发送催办通知

### 4. 转让历史追踪
- 记录资产的所有转让历史（原部门 → 新部门）
- 在资产详情页的流转历史中显示
