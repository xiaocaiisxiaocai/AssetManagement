## ❌ 失败的测试项（2项）

### 1. 登录后跳转 ❌

**现象**:
- 登录成功后跳转到 `/home` 而非预期的 `/dashboard/workspace`
- E2E 测试期望登录后直接进入工作台仪表盘

**分析**:
- 这是**路由配置问题**，非功能缺陷
- 实际系统设计：`/home` 就是首页路由（对应 `HomeWorkspace` 组件）
- 动态路由返回的第一个路由：
  ```json
  {
    "name": "Home",
    "path": "/home-root",
    "children": [{
      "name": "HomeWorkspace",
      "path": "/home",
      "component": "/dashboard/workspace/index"
    }]
  }
  ```

**结论**: ✅ **非BUG** — E2E测试脚本需要更新期望URL为 `/home`

---

### 2. 工作台仪表盘页面关键字检测失败 ❌

**现象**:
- 页面在15秒内未检测到关键字：`资产概览/资产总数/在库数/借出数/资产原值/逾期数`
- 测试截图：`debug-workspace-fail.png`

**可能原因**:
1. **数据加载延迟**：首次访问需要从后端获取统计数据
2. **空数据状态**：如果数据库中资产数量为0，可能显示空状态提示
3. **异步渲染**：Vue 组件需要时间渲染统计卡片
4. **E2E超时设置**：15秒可能不足以完成首次数据加载

**待验证**:
- 手动访问 `/home` 确认页面是否正常显示统计数据
- 检查浏览器控制台是否有API错误
- 延长E2E超时时间（改为30秒）

**临时解决方案**:
- 增加等待时间或使用更灵活的断言逻辑
