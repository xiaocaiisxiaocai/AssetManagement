## ❌ 失败的测试分析

### 失败类型 1: 流程完成判断
**影响测试**: 4-5 个  
**原因**: BPMN 模式下，流程推进逻辑不同
- 旧引擎：顺序推进，固定节点数
- BPMN 引擎：根据网关分支，动态推进

**示例**:
```csharp
// 旧代码：审批 3 次后流程完成
await Approve(flow.Id); // 节点 1
await Approve(flow.Id); // 节点 2  
await Approve(flow.Id); // 节点 3 -> 完成

// BPMN：取决于流程定义
await Approve(flow.Id, nodeId: "Task_supervisor");
// 可能推进到排他网关，根据条件走不同分支
// 审批次数不固定
```

**解决方案**: 
- 查询流程状态判断是否完成
- 不依赖固定的审批次数

### 失败类型 2: 空引用异常
**影响测试**: 2-3 个  
**原因**: 测试期望的数据结构不存在（如 flow.Data 为 null）

**解决方案**: 
- 添加 null 检查
- 验证 BPMN 流程是否正确初始化

### 失败类型 3: 资产详情测试
**测试**: `Asset_detail_returns_flows_and_recent_logs`  
**原因**: 流程未完成，导致关联数据为空

**解决方案**: 确保测试中流程正确完成
