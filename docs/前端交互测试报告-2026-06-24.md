## 🔍 问题分析

### 1. 表单字段选择器不匹配

**问题**: 
- `input[placeholder*="分类名称"]` 选择器在对话框中找不到匹配元素
- Element Plus 表单可能使用不同的 DOM 结构

**原因**:
- 前端组件可能使用自定义封装（`VbenForm`、`Schema` 等）
- placeholder 文本可能与预期不一致
- 输入框可能在 `el-form-item` 深层嵌套中

**解决方案**:
- 需要检查实际页面 DOM 结构
- 使用更通用的选择器（如 `.el-input__inner`）
- 或使用 `data-testid` 等测试专用属性

### 2. 对话框遮挡问题

**问题**:
- 新增分类对话框提交后未正确关闭
- 导致后续点击"编辑"按钮被遮挡（30秒超时）

**原因**:
- 表单验证失败（因字段未填充成功）
- 对话框未收到关闭信号
- 需要等待或手动关闭

**解决方案**:
- 提交前验证表单是否填充成功
- 失败时手动按 ESC 或点击取消关闭对话框
- 增加对话框关闭的显式等待

### 3. E2E 测试的局限性

**发现**:
- 自动化测试对前端组件封装敏感
- DOM 选择器维护成本高（组件升级易失效）
- 表单验证逻辑可能阻止提交

**建议**:
- 优先保证 **后端 API 测试覆盖** (已达 100%)
- E2E 测试专注于 **核心流程** (登录、列表加载、详情查看)
- **交互功能** 通过手工测试 + API 测试覆盖
